KEY FLOWS
─────────
① Sign-in:     Browser → /auth/start → Okta /v1/authorize → /auth/callback → Okta /v1/token (code exchange) → session cookie → /

⑤ Per-message: Browser POST /api/chat → FastAPI (verifies session id_token) → agent.py:
                 · Exchanges id_token for vaulted DB creds via Okta STS →③→ OPA Vault  (result cached per session)
                 · Runs Claude API agentic tool-use loop →⑥
                 · Each tool call: fetches creds from cache, POSTs to api.php with action + db creds →⑦→ MySQL
                 · All events streamed to browser as SSE  (session · token_exchange · text · tool_call · tool_result · end)

   Cache:       Vaulted secret cached server-side in memory (TTL ≈ 300 s).  Cache hit skips Okta STS call.
